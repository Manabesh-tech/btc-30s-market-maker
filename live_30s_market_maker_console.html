30s Live Quoting Console

Market-indicator-driven binary pricing for every new 30s bucket.

This prototype prices each side from a live fair probability, not from user flow. It is built for streaming microstructure signals every second, shrinking them toward neutral when confidence is low, and then applying a constant-edge payout model on top.

Core Pricing Formula

gross_multiplier = 1 - edge favored_payout = 100 * gross_multiplier / p - 100 other_payout = 100 * gross_multiplier / (1 - p) - 100 neutral(p = 0.50, edge = 4%) -> 92 / 92
Keep the model market-indicator-driven. Flow and inventory overlays are left out on purpose here. If we later decide they matter for diagnostics, we can study them without using them in the live quote.

Factor Explainer

  • Gross edge: your target hold before costs. A `4%` edge means you quote as if you want to keep `4` dollars per `100` staked if the mid is right.
  • PFOF / cost drag: the percentage you expect to give away after pricing. `Net keep = gross edge - PFOF`.
  • Top-5 notional imbalance: the main directional signal. Positive means the front of the book is more bid-heavy; negative means it is more offer-heavy.
  • Defaults are anchored to the strongest 7-day backtest we found: top-5 notional imbalance, `0.5` threshold, `3s` persistence, with triggered-state `P(up) ≈ 59.24%`.
  • Directional vs confidence factors: imbalance and persistence move the fair probability. Spread, thin depth, and jumpy vol mainly shrink confidence instead of pushing the quote UP or DOWN.
  • What is intentionally left out: this version is imbalance-first. Short-term flow confirmation is not moving the live quote by default until it proves itself out of sample.
  • What to review after launch: realized edge by state bucket: neutral, mild pressure, strong pressure, and high-vol shrink.
  • Persistence: how long that imbalance has lasted. The backtests liked pressure that stayed in place for about `3s`, not one noisy snapshot.
  • Microprice deviation: a small confirmation signal showing whether top-of-book weighting leans above or below the displayed mid.
  • Short-horizon vol: mainly a confidence control. Higher vol shrinks the quote back toward neutral because book pressure is less reliable in jumpy tape.
  • Thin depth: another confidence control. Thin books can exaggerate imbalance, so the model trusts the raw directional signal less when depth is weak.
  • Max favored P: a safety cap on how far the displayed probability can move away from `50/50`, even if the raw model gets more excited.
  • Quote cadence: how long each live quote stays fixed before the next refresh. This is aligned to exact wall-clock windows so routing uses one stable quote per interval.
  • Calibrate button: picks the strongest embedded backtest gate for the selected window and product, then writes those settings into the live gate and smoothing profile.
  • Today vs Last 7d: the switch changes the ranking metric. If both settings pick the same winner, that means the same depth and threshold combo was best in both windows.
  • Platform stream quote: this is Turboflow's actual public quote. The platform card shows their total implied edge plus the payout delta between our quote and theirs.
